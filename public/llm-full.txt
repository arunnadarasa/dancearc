# DanceArc

**DanceTech Protocol ‚Äî DanceArc MVP** ‚Äî reference hub and API on **Arc** with **native USDC**, **x402-style HTTP 402** challenges, and hooks for **Circle Gateway** (verify/settle) and **Nanopayments**.

## Stack

- **Frontend:** React 19, Vite 8, TypeScript
- **Chain:** Arc Testnet (`5042002`) via `viem/chains` [`arcTestnet`](https://github.com/wevm/viem/blob/main/src/chains/definitions/arcTestnet.ts)
- **Payments:** `402 Payment Required` + `X-Payment-Tx` (tx hash) + on-chain verification of **native USDC** transfers; optional `CIRCLE_API_KEY` ‚Üí Circle Gateway `POST /v1/gateway/v1/x402/verify`
- **x402:** HTTP `402` + JSON body (`x402Version`, `accepts[]`) aligned with the [x402](https://www.npmjs.com/package/x402) shape; Arc uses a custom `arc-native-usdc` accept block until the upstream schema lists Arc (`5042002`)
- **Circle:** [`@circle-fin/app-kit`](https://www.npmjs.com/package/@circle-fin/app-kit) for Bridge (see `/bridge`); [`@circle-fin/modular-wallets-core`](https://www.npmjs.com/package/@circle-fin/modular-wallets-core) for Modular Wallets (see [Web SDK](https://developers.circle.com/w3s/modular-wallets-web-sdk))
- **Wallet UI:** `InjectedWalletBar` uses **EIP-1193** (`window.ethereum`) for **native USDC** transfers on Arc. **`CircleModularPanel`** calls `toModularTransport` + `createPublicClient` to verify your **Client Key** against `https://modular-sdk.circle.com` (override with `VITE_CIRCLE_MODULAR_CLIENT_URL`). Passkey login and smart-account sends follow [Modular Wallets Web SDK](https://developers.circle.com/w3s/modular-wallets-web-sdk).

## Quick start

```bash
npm install
cp .env.example .env
# Terminal 1
npm run server
# Terminal 2
npm run dev
```

Open `http://localhost:5173` ‚Äî **Judge score (live)** at `/dance-extras` demonstrates pay-per-call on Arc.

### Local deployment (production build + API)

After `npm run build`, serve the static app and the API together:

```bash
npm run start:local
```

This runs Express on **8787** and `vite preview` on **5173** with `/api` proxied (same as dev). Open `http://127.0.0.1:5173`. Use **Testnet** keys only; never commit `.env`.

## Environment

Create keys in **[Circle Console](https://console.circle.com/api-keys/create)** (use **Testnet** for hackathon). For an **API Key**, under **Applicable platforms ‚Üí Web**, add an **Allowed domain** of `localhost` (omit the port ‚Äî e.g. use `localhost`, not `localhost:5173`) so local Vite (`npm run dev`) can call Circle APIs from the browser when required. Add your production host when you deploy.

3. Run:

```bash
npm run burst
```

4. Paste explorer links or CSV of hashes for submission **transaction frequency** evidence.

## Margin story (why not ‚Äúnormal‚Äù gas?)

On many networks, **fixed per-tx overhead** in ETH or an unstable gas token destroys **sub-cent, per-event** pricing: you either **batch** (latency, custody) or **overcharge** (bad UX). **Arc + USDC-denominated gas + micro-native transfers** align the **fee unit** with the **business unit** (USDC) so high-frequency, low-dollar APIs and agents can settle **per action** without a subscription wrapper.

# Hackathon checklist (Agentic Economy on Arc)

## Tracks

This MVP aligns with **Per-API monetization** and **Usage-based compute billing**: paid `POST /api/judges/score` and `/api/dance-extras/live/judge-score/testnet` with **per-action pricing** (default **0.001 USDC**, configurable via `PER_ACTION_USDC`, must stay **‚â§ $0.01**).

## Economic proof

1. **Per-action pricing:** `PER_ACTION_USDC` (default `0.001`) ‚Äî see `GET /api/health`.
2. **Transaction frequency:** run `npm run burst` with a funded test key to produce **50+** Arc Testnet txs; hashes print to stdout (link recipient on ArcScan).
3. **Margin vs traditional gas:** On general-purpose L1s, a single paid interaction often implies **base fee + priority fee** in a volatile gas token, frequently **orders of magnitude above** sub-cent unit economics unless you batch, subsidize, or custody. **Arc uses USDC for gas** and supports **high-frequency microflows** (Circle Nanopayments + Gateway x402) so **per-call** pricing can match **actual usage** without forcing subscription bundling.
